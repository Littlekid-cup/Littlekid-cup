    # Git 基本操作

    • git init：初始化仓库

    • git status: 查看文件状态（红色文件：在工作区;绿色文件:在暂存区）

    • git add+文件名（文件夹名）：添加文件（文件夹）到暂存区

      git add --all ：添加所有文件到暂存区

      git add . ：将所有当前文件添加到暂存区

    • git commit：将文件从暂存区添加到仓库区

      git commit -m '提交说明（描述用）'

    • git config --global user.email "填写邮箱"

      git config --global user.name "填写用户名"

    • git log: 提交日志

    • git reset: 版本回退，重置

      git reset --hard +版本号：回到某一版本（git log 查看 commit）

    # 新建.gitignore 文件存放不想上传文件的文件名

    ！！.gitignore 配置文件自己不需要被忽略

    # git 分支操作

    • git branch ：查看所有的分支（绿色代表当前分支）

    • git branch +分支名：创建分支

    • git checkout +分支名：切换分支

    • git merge +分支名：合并分支

      先切换到主分支，再使用 git merge+分支名合并分支

    • git branch -d +分支名：删除分支

    • git checkout -b +分支名：创建并切换分支

    • git push -u origin +分支名：推送给远程仓库的某个分支，-u 表示将其设置为默认

    # 分支合并解决冲突

    • 保留双方文件 ( git pull 基本上等于 git fetch 加上 git merge)
    • git add .并且 git commit -m，提交
    • 1.git checkout dev 切换到 dev 分支
    • 2.git pull origin dev 把远程 dev 分支更新的代码拉取到本地
    • 3.git merge feature/xxx 把自己的分支合并到 dev 分支上，会冲突，解决冲突，完成后重新提交代码推到远程 dev 分支。
    • 4.进入 vim 环境使用 “：q”可以退出
    • 5.所有人可以 git pull origin dev ,保证所有人的 dev 分支是一致的

    # git 远程仓库

    • git clone 从远程仓库中克隆代码到本地

    • git push 把自己的数据更新到远程仓库中

    • git pull 拉取远程仓库的更新

    • git remote add 仓库别名 仓库地址

      （默认仓库名:origin 注意克隆的才有 origin）

      git remote add origin master

    ​ git remote -v 查询

    ​ git remote -u origin master(设置默认地址)

    填写仓库名称，路径，归属，私有，创建
